# システム管理画面（プラットフォーム運営者向け）の実装計画

ご指示いただいた表示項目をすべて盛り込んだ、システム管理画面の実装計画を作成しました。

## > [!IMPORTANT]
## User Review Required
各項目の計算ロジックに間違いがないか、以下の内容をご確認ください。

### 表示・計算項目リスト
管理画面の一覧表（テーブル）には、町内会ごとに以下の項目を表示します。

1. **町内会・自治会の名称**
2. **全世帯数**（町内会が申告している対象世帯数）
3. **実登録世帯数**（実際にLINE連携して登録された会員数）
4. **代表者名**
5. **月額世帯単価**（例：登録1世帯につき〇円。不要な場合は0円設定可能とします）
6. **プッシュ送信単価**（例：3円）
7. **原価**（例：1.5円）
8. **月額請求額**
   * **計算式**: `(実登録世帯数 × 月額世帯単価) ＋ {(当月のプッシュ送信数 － 無料枠200通) × プッシュ送信単価}`
   * （※プッシュ送信数が200通以下の場合はプッシュ料金は0円として計算します）
9. **納入済金額**（運営者が「今月分としていくら入金されたか」を手動入力して保存できる項目）
10. **（参考）運営利益（粗利）** ※画面上に表示
   * **計算式**: `月額請求額 － {(当月のプッシュ送信数 － 無料枠200通) × 原価}`

## Proposed Changes

### 1. データベースの改修 (SQL実行)
システム管理に必要なカラム・テーブルを追加します。

#### [MODIFY] `system_settings` テーブル
以下の設定カラムを追加します。
*   `push_cost_price` (プッシュ送信原価, 初期値 1.5)
*   `monthly_household_price` (月額世帯単価, 初期値 0 または指定値)

#### [NEW] `platform_payments` テーブル（新設）
運営側が町内会からの「納入済金額」を月ごとに管理するためのテーブルを新設します。
*   `neighborhood_id` (町内会ID)
*   `target_month` (対象月, 例: "2026-05")
*   `paid_amount` (納入済金額)

### 2. システム管理画面の開発
#### [NEW] `src/app/system/page.tsx`
*   システム管理者専用の画面（パスワード保護などをかけることも可能ですが、今回はまず専用URLとして作成します）。
*   画面を開いた月（当月）のデータを基準に、上記の1〜10の項目を一覧表示するダッシュボードテーブルを実装します。
*   各町内会の「月額世帯単価」「プッシュ送信単価」「原価」および「納入済金額」をこの画面から直接入力・保存できるようにします。

### 3. 自治会役員画面の改修
#### [MODIFY] `src/components/AdminView.tsx`
*   現在のダッシュボード上部にある「未納会費の件数」という部分を、町内会側から見た**「プラットフォームへの今月のシステム利用料 予想請求額」**に表示を変更します。

## Verification Plan
*   `/system` にアクセスし、すべての町内会がリスト表示されること。
*   月額世帯単価、送信単価、原価を入力すると、即座に「月額請求額」と「利益」が再計算されること。
*   納入済金額を入力し、保存できること。
