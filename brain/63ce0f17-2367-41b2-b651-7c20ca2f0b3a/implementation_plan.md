# タブ分割によるメニュー構成の最適化プラン

「タブを作って分ける」というアイデア、素晴らしいです！スマートフォンの狭い画面でも整理して見せることができる最適なアプローチです。

ご要望を踏まえ、**「画面下のタブ（親）」と「画面上のタブ（子）」の2階層に分ける**というUI設計をご提案します。

## Proposed Changes

### 1. 画面下部（ボトムナビゲーション）の整理
現在の画面下のメニューを以下の**5つ**に整理し、1つ新しいタブを作成します。

1. **ホーム**（アプリを開いた最初の画面）
2. **連絡**
3. **回覧板**
4. **イベント**
5. **各種機能**（または「町内会機能」） ⬅️ **[NEW] ここに新しいタブを作成します**

※画面下のタブは5つであれば、スマホでもギリギリ押しやすいサイズで綺麗に収まります。

### 2. 「各種機能」タブの中身を「上のタブ」で分割
新しく作った画面下の「各種機能」タブをタップして開いた画面の中に、**画面上部の切り替えタブ**を作成し、そこで画面を分割します。

以下のように、上部のタブをタップして表示内容を切り替える仕組みです。

```text
======================================
 [ 会費 ]  [ Web会議 ]  [ 総会 ]   <-- 画面上部のタブ（ここで分ける）
======================================

（選択したタブの中身がここに表示される）

```

この設計にすることで、以下のようなメリットがあります。
*   画面下のボタンが小さくなりすぎず、押し間違いを防げる
*   今後「ゴミカレンダー」などさらに機能が増えても、上のタブを横にスクロールできるようにすれば無限に追加できる
