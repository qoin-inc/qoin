# 映えel-town の改善と地図（マイel-town）連携の実装

## Goal
「映えel-town」にあったイベントカレンダー機能・イベント日時入力機能を廃止し、「食べel-town」と同様に「場所（位置情報）」ベースの投稿に統一します。
また、日本地図（マイel-town）上には「その町内会の最新の投稿」をピンとして表示し、ピンをタップした際にその町内会の全投稿一覧が確認できる機能を追加します。

## User Review Required
> [!IMPORTANT]
> - イベントカレンダーを完全に削除し、映えel-townも「タイムライン表示」のみとなります。
> - 投稿時には「イベント日」ではなく「場所」を入力する形に統一されます。
> - 地図でピンをタップした際、「地図の下部」に引き上げ式でその町内会の投稿リストが表示されるUIで進めてよろしいでしょうか？（LINEの仕様に近いスライド式のリストを想定しています）

## Proposed Changes

### app/portal/page.tsx
- **[MODIFY]** カレンダー関連のState（`sightViewMode`, `currentDate`, `selectedDate`）およびカレンダー描画関数（`renderCalendar`）を削除。
- **[MODIFY]** 投稿作成時のモーダルで、`sight` カテゴリの際に表示していた「イベント日時」入力フィールドを削除し、「場所」入力フィールドを表示するよう変更。
- **[MODIFY]** `fetchData` メソッドにて、地図用の `towns` リストを生成する際、常に「最新の投稿」がピン情報として残るようにロジックを修正。
- **[MODIFY]** 新規State `selectedTownId` を追加し、地図タブ（マイel-town）を開いている際に特定のピンをタップした際、その町内会に絞り込んだ「投稿一覧リスト」を画面下部にスライド表示するUIを実装。

### components/MapComponent.tsx
- **[MODIFY]** ピンのポップアップに表示される「観れ」という表記を「映え」に修正。
- **[MODIFY]** ポップアップ内のボタンをタップした際に親コンポーネント（page.tsx）へイベントを渡し、一覧リストが表示されるように連携。

## Verification Plan
- カレンダーが表示されないこと、イベント日時の入力がなくなり場所が入力できることを確認。
- 過去のイベント日時データが画面崩れを起こさないことを確認。
- 地図タブ（マイel-town）でピンをタップすると、その町内会の最新情報と過去の投稿一覧が表示されることを確認。
