# 実装完了レポート（Day 3：電子回覧板・イベントの分割とファイル添付）

ご要望いただいた機能拡張の実装が完了いたしました！
単一の「お知らせ」だった機能を、「情報連絡」「電子回覧板」「イベント通知」の３つに拡張し、さらにPDFなどのファイル添付とイベント参加申し込みができるようにアップグレードしました。

## 実装内容まとめ

### 1. 役員画面（配信の新規作成と一覧）
- 新規作成時に、「電子回覧板」「情報連絡」「イベント通知」のどの種類で配信するかを選べるラジオボタンを追加しました。
- 「電子回覧板」または「イベント通知」を選んだ場合のみ、画面に**「ファイル添付」ボタン**が出現します。ここにPDF、Word、Excelなどを指定して送信すると、自動でクラウドの安全な領域（Supabase Storage）にアップロードされます。
- 一覧画面には「回覧板」「イベント」のようなバッジが付き、イベントについては「既読率」だけでなく「参加申込人数」も確認できるようになりました。

### 2. 住民画面（3つのタブと申込ボタン・参加人数指定）
- 画面上部にタブ（［すべて］［連絡］［回覧板］［イベント］）を追加し、見たい情報を絞り込めるようになりました。
- 詳細画面を開いたとき、ファイルが添付されていれば「添付ファイルを開く」ボタンが表示され、タップするだけで内容を表示・ダウンロードできます。
- 「イベント通知」の場合、画面下部に専用のアクションが用意されます：
  - **「このイベントに申し込む」ボタン**（参加する場合）
    - ※タップすると**大人・子供の人数を個別で設定するフォーム**が開き、具体的な人数を通知できるようになりました！
  - **「内容は確認した（不参加）」ボタン**（今回は参加しないが既読にする場合）

---

> [!WARNING]
> **※動作確認の前に、必ず以下のデータベース更新を行ってください※**
> 
> 新しい機能を使うために、Supabaseのデータベースに設定を追加する必要があります。
> お客様のPC内に新しく生成したファイル：`Day3_update.sql`（または以下のSQLコード）をコピーし、Supabaseダッシュボードの「SQL Editor」に貼り付けて「RUN（実行）」を行ってください。

```sql
-- 1. `circulars` テーブルへのカラム追加
ALTER TABLE public.circulars
ADD COLUMN IF NOT EXISTS category TEXT DEFAULT 'info' CHECK (category IN ('circular', 'info', 'event')),
ADD COLUMN IF NOT EXISTS attachments JSONB DEFAULT '[]'::jsonb;

-- 既存のデータを 'info' として更新
UPDATE public.circulars SET category = 'info' WHERE category IS NULL;

-- 2. イベント申込を管理する `event_applications` テーブルの新規作成
CREATE TABLE IF NOT EXISTS public.event_applications (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    circular_id BIGINT REFERENCES public.circulars(id) ON DELETE CASCADE,
    resident_name TEXT NOT NULL,
    applied_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 3. ファイルを保存する Storage (添付ファイル保管庫) の設定
INSERT INTO storage.buckets (id, name, public) 
VALUES ('attachments', 'attachments', true)
ON CONFLICT (id) DO NOTHING;

-- Storageへのアクセス権（ポリシー）の作成
DO $$ 
BEGIN
  CREATE POLICY "Allow public viewing" 
  ON storage.objects FOR SELECT 
  USING (bucket_id = 'attachments');
EXCEPTION WHEN duplicate_object THEN null;
END $$;

DO $$ 
BEGIN
  CREATE POLICY "Allow all uploads" 
  ON storage.objects FOR INSERT 
  WITH CHECK (bucket_id = 'attachments');
EXCEPTION WHEN duplicate_object THEN null;
END $$;
```

## 実際に動かしてみましょう！

データベースの更新（上記SQLの実行）が終わりましたら、Next.jsのアプリ（`npm run dev`）で以下の検証を行ってみてください。

1. **イベントを作ってみる**：役員画面から「イベント通知」を選び、適当なPDF等のファイルを選んで配信する。
2. **申し込んでみる**：住民側の画面でタブを切り替え、イベントを開き「このイベントに申し込む」を押す。
3. **役員画面に戻って確認**：役員画面のリストで、申込者の数が「1人」となっていることを確認する。
