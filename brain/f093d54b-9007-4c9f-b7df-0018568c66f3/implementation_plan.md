# [Goal Description]

「ResidentView」のUIデザインを刷新し、現在の「LINE風チャットUI」から、視認性と操作性に優れた「モダンアプリUI」へ変更します。最新の情報が一番上に表示され、必要なメニューにいつでもワンタップでアクセスできる、使いやすく美しいデザインを実現します。

## User Review Required

> [!IMPORTANT]
> **デザインの方向性についてご確認をお願いします**
> 
> これまでのLINE風の表示（下から上に新しいものが並ぶフキダシ）をやめ、以下の大きく2点の変更を行ってよろしいでしょうか？
> 
> 1. **投稿の表示順とデザイン**
>    - 新しいお知らせが「一番上」にくるように戻します（標準的なニュースアプリやブログと同じ形）。
>    - フキダシではなく、読みやすさ重視の「カード型（白背景にきれいな影がついた四角いデザイン）」に変更し、添付画像なども大きく見やすく配置します。
> 2. **メニューの操作性向上**
>    - 開閉式の「リッチメニュー」を廃止します。
>    - 代わりに、画面の一番下に常に表示される「ボトムナビゲーション（ホーム、回覧板、連絡、イベント などが並ぶバー）」を採用します（InstagramやTwitterなどのアプリの下にあるようなメニューです）。
>    - 町内会費などの細かいメニューは、ホーム画面内に「タイル」として見やすく並べます。

## Proposed Changes

### ResidentView (フロントエンド)

#### [MODIFY] ResidentView.tsx
- **ロジックの修正**:
  - `filteredCirculars.reverse()` を削除し、取得したデータ（最新順）をそのまま上から下にレンダリングする。
  - 自動で一番下にスクロールする処理を削除し、常にページトップから見られるようにする。
- **カード型UIへの変更**:
  - フキダシのデザイン（三角形のしっぽ等）を削除。
  - 各連絡事項を、角丸でドロップシャドウのついたクリーンな「カード」として再実装。
  - カテゴリバッジ（回覧板、イベントなど）や日付をカード上部に配置し、タイトルを大きく強調。
- **ボトムナビゲーションの実装**:
  - `isMenuOpen` やトグルバーのロジックを削除。
  - 画面最下部に固定された高さ（例: 60px〜80px）の `BottomNavigation` を実装し、主要な4つのタブ（ホーム、回覧板、連絡、イベント）を配置。
  - タップしているタブの色がアクティブになる（例: qoin-main色）フィードバックを追加。
- **ホーム画面の拡充**:
  - 「ホーム」タブを開いた際に、重要なお知らせだけでなく、各種サブメニュー（町内会費、オンライン総会など）へのショートカットボタン（タイル状のデザイン）を表示する。
